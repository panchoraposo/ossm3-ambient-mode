# UC20-T6: Traffic Mirroring

## Objective

Demonstrate that the **waypoint proxy** can mirror (shadow) live traffic to a secondary destination via `HTTPRoute` (Gateway API). All primary traffic goes to `reviews` (all versions), while an identical copy is sent to a specific version (`reviews-v2`) for observation — without affecting the responses seen by the caller.

## Prerequisites

- Both clusters running with bookinfo deployed and accessible
- `generate-traffic.sh` running for Kiali visualization (recommended — shows mirrored traffic reaching v2)
- Kiali open (OSSMC via ACM console):
  https://console-openshift-console.apps.cluster-72nh2.dynamic.redhatworkshops.io/ossmconsole/graph

## Quick Run

```bash
./ossm/uc20-t6-verify.sh
```

## Architecture

Traffic mirroring duplicates requests at the waypoint level:

```
                        ┌──→ reviews (primary, response returned to caller)
productpage → waypoint ─┤
                        └──→ reviews-v2-mirror (shadow copy, fire-and-forget)
```

- The caller (`productpage`) sees the normal response from `reviews`
- `reviews-v2-mirror` receives an identical copy of every request but its response is discarded
- Mirroring is fire-and-forget: failures in the mirror target do not affect the primary flow

This is useful for testing new versions with real traffic, debugging, or feeding analytics pipelines — all without impacting production traffic.

### HTTPRoute requestMirror filter

```yaml
filters:
- type: RequestMirror
  requestMirror:
    backendRef:
      name: reviews-v2-mirror
      port: 9080
```

This is a **Standard** (GA) feature in Gateway API since v1.3.0.

## Manual Steps

### 1. Create version-specific service for the mirror target

```bash
oc --context east apply -f - <<EOF
apiVersion: v1
kind: Service
metadata:
  name: reviews-v2-mirror
  namespace: bookinfo
spec:
  ports:
  - port: 9080
    name: http
  selector:
    app: reviews
    version: v2
EOF
```

### 2. Verify baseline — traffic goes to all versions

```bash
PRODUCTPAGE_POD=$(oc --context east get pods -n bookinfo -l app=productpage -o jsonpath='{.items[0].metadata.name}')
oc --context east exec -n bookinfo "$PRODUCTPAGE_POD" -- python3 -c "
import urllib.request, json, time
for i in range(6):
    time.sleep(1)
    req = urllib.request.Request('http://reviews:9080/reviews/0')
    try:
        with urllib.request.urlopen(req, timeout=10) as resp:
            pod = json.loads(resp.read().decode()).get('podname', '?')
            print(f'  [{i+1}] {pod}')
    except Exception as e:
        print(f'  [{i+1}] error: {type(e).__name__}')
"
```

Expected: mixed responses from v1, v2, v3 (round-robin).

### 3. Deploy reviews-waypoint (L7 proxy)

The HTTPRoute with `requestMirror` requires a waypoint proxy. Deploy it on demand:

```bash
oc --context east apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: reviews-waypoint
  namespace: bookinfo
  labels:
    istio.io/waypoint-for: service
spec:
  gatewayClassName: istio-waypoint
  listeners:
    - name: mesh
      port: 15008
      protocol: HBONE
EOF

oc --context east label svc reviews -n bookinfo istio.io/use-waypoint=reviews-waypoint --overwrite
oc --context east wait --for=condition=Ready pod -l gateway.networking.k8s.io/gateway-name=reviews-waypoint -n bookinfo --timeout=60s
```

### 4. Apply HTTPRoute with requestMirror filter

```bash
oc --context east apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: reviews-mirror
  namespace: bookinfo
spec:
  parentRefs:
  - kind: Service
    group: ""
    name: reviews
    port: 9080
  rules:
  - filters:
    - type: RequestMirror
      requestMirror:
        backendRef:
          name: reviews-v2-mirror
          port: 9080
    backendRefs:
    - name: reviews
      port: 9080
EOF
```

### 5. Send traffic and verify mirroring

Send requests and check the waypoint stats to confirm mirrored traffic reaches reviews-v2:

```bash
# Send 8 requests
for i in $(seq 1 8); do
  curl -s -o /dev/null -w "%{http_code} " -m 10 \
    "http://bookinfo.apps.cluster-64k4b.64k4b.sandbox5146.opentlc.com/productpage"
  sleep 1
done
echo ""

# Check mirror stats in waypoint
REVIEWS_WP=$(oc --context east get pod -n bookinfo \
  -l gateway.networking.k8s.io/gateway-name=reviews-waypoint \
  -o jsonpath='{.items[0].metadata.name}')

oc --context east exec "$REVIEWS_WP" -n bookinfo -- \
  pilot-agent request GET clusters 2>/dev/null \
  | grep "reviews-v2-mirror" | grep "rq_total"
```

Expected: `rq_total` > 0, confirming mirrored requests reached reviews-v2.

### 6. Verify via ztunnel logs

```bash
ZTUNNEL_POD=$(oc --context east get pods -n ztunnel -l app=ztunnel -o jsonpath='{.items[0].metadata.name}')

echo "Connections waypoint → v2-mirror (shadow):"
oc --context east logs "$ZTUNNEL_POD" -n ztunnel --tail=60 | grep "reviews-v2-mirror" | wc -l
```

Expected: connections > 0 confirming the waypoint sends shadow copies to v2.

### 7. Cleanup

```bash
oc --context east delete httproute reviews-mirror -n bookinfo
oc --context east delete svc reviews-v2-mirror -n bookinfo
oc --context east label svc reviews -n bookinfo istio.io/use-waypoint-
oc --context east delete gateway reviews-waypoint -n bookinfo
```

### 8. Verify recovery

```bash
curl -s -o /dev/null -w "EAST: HTTP %{http_code}\n" http://bookinfo.apps.cluster-64k4b.64k4b.sandbox5146.opentlc.com/productpage
```

Expected: HTTP 200, normal latency.

## Expected Results

| Phase | reviews (primary) | reviews-v2-mirror (shadow) |
|-------|-------------------|---------------------------|
| Baseline | receives traffic (all versions) | does not exist |
| Mirror active | receives primary traffic | **receives shadow copies** (fire-and-forget) |
| After cleanup | receives traffic (all versions) | service deleted |

> **Alternativa VirtualService**: El mirroring también puede configurarse con `VirtualService` (`mirror` + `mirrorPercentage`) combinado con `DestinationRule` (subsets). VirtualService es Technology Preview en OSSM 3.2 ambient mode.

## What is Service Mesh here

| Component | Role | Mesh feature? |
|-----------|------|:------------:|
| HTTPRoute (requestMirror) | Configures traffic mirroring to reviews-v2 | Yes — traffic management (GA) |
| reviews-v2-mirror (Service) | Version-specific routing target for mirror | No — standard K8s |
| reviews-waypoint | Duplicates requests: sends primary + shadow copy | Yes — L7 data plane |
| ztunnel | Routes traffic through waypoints (HBONE) | Yes — L4 data plane |
| istiod | Pushes HTTPRoute config to waypoints | Yes — control plane |

## Key Takeaway

Traffic mirroring in ambient mode is enforced by the **waypoint proxy** via `HTTPRoute` (Gateway API). The waypoint duplicates every incoming request: the primary copy goes to the configured backend and the caller receives its response, while a shadow copy is sent fire-and-forget to the mirror target. Using `HTTPRoute` with the `requestMirror` filter (Standard/GA since Gateway API v1.3.0), this feature is fully supported without depending on Istio-specific CRDs. This enables safe testing of new service versions with real production traffic — failures or slowness in the mirror target have no impact on the caller.
