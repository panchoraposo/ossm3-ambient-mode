# UC11: Special Ciphers (Legacy TLS Origination) — Ambient (OSSM 3.2)

## Objective

Demostrar cómo **OSSM 3.2 (Istio Ambient Mode)** puede mantener el mesh **moderno y seguro** y, a la vez, permitir conectividad hacia un backend externo/legacy que requiere **TLS 1.0 + cipher suites antiguas**, **aislando** la lógica legacy al camino de egress.

Además, replicar el patrón “tipo 3scale” donde un **header especial** habilita una ruta “compat” **solo** para ese destino (sin bajar la postura de seguridad global del mesh).

## Qué muestra este caso

- Un backend “legacy” que **solo acepta TLS 1.0** y un cipher legacy (por defecto: `ECDHE-RSA-AES256-SHA`)
- Un **egress waypoint** (ambient) que hace enforcement L7 (routing por header) y centraliza el egress
- **Header-based downgrade**:
  - sin header → ruta “modern/strict” → falla
  - con header → ruta “compat” → funciona
- Validación en logs del servidor TLS (se observa **TLS 1.0 Handshake** y el cipher)

## Arquitectura (alto nivel)

```
workload (mesh) ──HTTP──> ztunnel ──HBONE/mTLS──> egress waypoint (Envoy)
                                         │
                                         └─ L7 routing por header
                                              ├─ strict connector  → FAIL (fuerza TLS >=1.2 hacia backend)
                                              └─ compat connector  → OK   (origina TLS1.0 + legacy cipher)
```

**Importante**: el workload habla **HTTP** hacia el host externo “registrado” con `ServiceEntry`. El origination TLS legacy se realiza en “connectors” dedicados (HAProxy) para evitar depender de campos no expuestos en `DestinationRule` en OSSM 3.2.

## Quick Run (Istio-only)

```bash
./ossm/uc11-verify.sh east
```

Para WEST:

```bash
./ossm/uc11-verify.sh west
```

Variables útiles:

- `AUTO_ENABLE_DNS_CAPTURE=true`: auto-habilita DNS capture si está apagado
- `KEEP_RESOURCES_ON_FAIL=true`: deja recursos para inspección
- `NO_PAUSE=true`: modo no interactivo
- `KIALI_DEMO=true`: despliega un generador de tráfico en background y **no** limpia recursos (para mostrar en Kiali)
- `TRAFFIC_NS=bookinfo`: namespace donde corre el generador (por defecto: `bookinfo`)
- `TRAFFIC_PERIOD_SEC=2`: intervalo entre requests del generador
- `BUILD_MODE=binary|git|prebuilt`: cómo se construyen/obtienen imágenes (por defecto: `binary`)
  - `binary`: build subiendo directorio local (permite probar sin pushear cambios)
  - `git`: build en-cluster desde Git (sin “Uploading directory ...”, requiere que el repo remoto tenga `ossm/uc11/...`)
  - `prebuilt`: no build; usa `LEGACY_TLS_SERVER_IMAGE` y `HAPROXY_CONNECTOR_IMAGE`
- `IMAGE_BUILD_NS=uc-images`: namespace **no ambient** donde corren los Builds (evita que aparezcan en Kiali)
- `GIT_CONTEXT_LEGACY_TLS=ossm/uc11/legacy-tls-server`: contextDir a usar cuando `BUILD_MODE=git`
- `GIT_CONTEXT_HAPROXY=ossm/uc11/legacy-haproxy-connector`: contextDir a usar cuando `BUILD_MODE=git`

## Connectivity Link (Kuadrant) value-add

Construye sobre lo anterior y demuestra valor agregado:

- En Istio puro, el “downgrade” depende del cliente (header).
- Con Connectivity Link, se puede **controlar quién** accede al flujo “legacy”, exigiendo **API key** en el edge y habilitando el downgrade solo para requests autorizadas (vía `HTTPRoute` + `AuthPolicy`).

### Quick Run (Kuadrant)

```bash
./ossm/uc11-kuadrant-verify.sh east
```

Expected:

- `GET /modern` → falla (no downgrade)
- `GET /legacy` sin API key → **401**
- `GET /legacy` con API key → **200**

## Demo en Kiali (pasos)

1) Ejecuta UC11 en modo demo:

```bash
NO_PAUSE=true KIALI_DEMO=true ./ossm/uc11-verify.sh east
```

2) Abre Kiali y mira el **Graph** (Workload graph) para estos namespaces:

- `${TRAFFIC_NS}` (cliente/generador; por defecto `bookinfo`)
- `egress-legacy` (waypoint + connectors)
- `legacy-backend` (backend TLS legacy)

3) Espera 1–2 minutos y selecciona rango “Last 1m/5m”. Deberías ver:

- tráfico alternando hacia **strict connector** (errores) y **compat connector** (OK)
- el waypoint como punto de control de egress

### Demo en Kiali (Kuadrant)

Ejecuta el escenario Kuadrant en modo demo (deja recursos y genera tráfico hacia el Gateway):

```bash
NO_PAUSE=true KIALI_DEMO=true ./ossm/uc11-kuadrant-verify.sh east
```

En Kiali, revisa el **Graph** para estos namespaces:

- `${TRAFFIC_NS}` (cliente/generador; por defecto `bookinfo`)
- `api-gateway` (Gateway + enforcement Kuadrant)
- `legacy-probe` (servicio mesh que llama al destino legacy)
- `egress-legacy` (waypoint + connectors)
- `legacy-backend` (backend TLS legacy)

Deberías ver tráfico a:

- `/modern` (fallos porque no se habilita downgrade)
- `/legacy` sin auth (401)
- `/legacy` con API key (200)
