# UC20-T5: Egress Control

## Objective

Demonstrate that ambient mode can **control egress traffic** (outbound to external services) using an egress gateway pattern: a dedicated waypoint proxy in a separate namespace processes all external traffic, enabling `AuthorizationPolicy` enforcement, observability, and TLS origination.

## Prerequisites

- Both clusters running with bookinfo deployed and accessible
- **DNS capture enabled** in IstioCNI CR (`cni.ambient.dnsCapture: true`)
- `generate-traffic.sh` running for Kiali visualization (recommended)
- Kiali open (OSSMC via ACM console):
  https://console-openshift-console.apps.cluster-72nh2.dynamic.redhatworkshops.io/ossmconsole/graph

> **Note**: `ServiceEntry` is GA in OSSM 3.2. DNS capture is required for ztunnel to intercept external traffic.

## Quick Run

```bash
./ossm/uc20-t5-verify.sh
```

## Architecture

Egress control in ambient mode requires a different approach than sidecar mode. Without sidecars, ztunnel only intercepts traffic to known mesh services. DNS capture bridges this gap:

```
                     DNS query                     ztunnel
productpage ──────────────────→ ztunnel DNS proxy ──→ resolves httpbin.org to VIP 240.240.0.x
                                                      (instead of real external IP)

                     data path
productpage ──→ ztunnel ──→ egress-gateway (waypoint) ──→ httpbin.org (real IP)
                  │              │
                  │              └─ AuthorizationPolicy enforced here (L7)
                  └─ HBONE tunnel to waypoint VIP
```

Key components:
- **DNS capture** (IstioCNI): ztunnel intercepts DNS queries, resolves ServiceEntry hosts to auto-allocated VIPs
- **ServiceEntry**: registers external hosts in the mesh registry with auto-allocated VIPs
- **Egress waypoint**: a waypoint proxy in a dedicated namespace (`egress-control`) acts as the egress gateway
- **AuthorizationPolicy**: L7 access control on the egress gateway (methods, paths, source identities)

Without DNS capture, external traffic bypasses ztunnel entirely (resolves to real IPs, no iptables match).

## Manual Steps

### 0. Verify DNS capture is enabled

```bash
oc --context east get istiocni default -o jsonpath='{.spec.values.cni.ambient.dnsCapture}'
# Expected: true
```

If `false`, enable it:

```bash
oc --context east patch istiocni default --type merge \
  -p '{"spec":{"values":{"cni":{"ambient":{"dnsCapture":true}}}}}'

# Restart istio-cni-node, ztunnel, and bookinfo pods to pick up new iptables rules
oc --context east rollout restart ds/istio-cni-node -n istio-cni
oc --context east rollout restart ds/ztunnel -n ztunnel
oc --context east rollout restart deploy -n bookinfo
```

### 1. Verify baseline — external access without control

```bash
PRODUCTPAGE_POD=$(oc --context east get pods -n bookinfo -l app=productpage -o jsonpath='{.items[0].metadata.name}')
oc --context east exec -n bookinfo "$PRODUCTPAGE_POD" -- python3 -c "
import urllib.request
req = urllib.request.Request('http://httpbin.org/get')
with urllib.request.urlopen(req, timeout=10) as resp:
    print(f'Status: {resp.status}')
    print(f'Server: {resp.headers.get(\"server\", \"unknown\")}')
"
```

Expected: Status 200, Server: gunicorn (direct, no mesh interception).

### 2. Create egress namespace with waypoint

```bash
oc --context east create namespace egress-control
oc --context east label namespace egress-control istio.io/dataplane-mode=ambient
istioctl --context east waypoint apply --enroll-namespace --name egress-gateway --namespace egress-control
```

### 3. Create ServiceEntry for httpbin.org

```bash
oc --context east apply -f - <<EOF
apiVersion: networking.istio.io/v1
kind: ServiceEntry
metadata:
  name: httpbin-org
  namespace: egress-control
spec:
  hosts:
  - httpbin.org
  ports:
  - number: 80
    name: http
    protocol: HTTP
  resolution: DNS
EOF
```

Verify the ServiceEntry is bound to the waypoint:

```bash
oc --context east get serviceentry httpbin-org -n egress-control -o yaml | grep -A 10 "status:"
# Expected: WaypointBound: True, address: 240.240.0.x
```

### 4. Verify traffic goes through egress gateway

```bash
oc --context east exec -n bookinfo "$PRODUCTPAGE_POD" -- python3 -c "
import urllib.request
req = urllib.request.Request('http://httpbin.org/get')
with urllib.request.urlopen(req, timeout=10) as resp:
    print(f'Status: {resp.status}')
    print(f'Server: {resp.headers.get(\"server\", \"unknown\")}')
    print(f'x-envoy-upstream-service-time: {resp.headers.get(\"x-envoy-upstream-service-time\", \"none\")}')
"
```

Expected: Status 200, **Server: istio-envoy** (traffic went through the egress gateway).

### 5. Apply AuthorizationPolicy — only allow GET /get

```bash
oc --context east apply -f - <<EOF
apiVersion: security.istio.io/v1
kind: AuthorizationPolicy
metadata:
  name: httpbin-egress-policy
  namespace: egress-control
spec:
  targetRefs:
  - kind: ServiceEntry
    group: networking.istio.io
    name: httpbin-org
  action: ALLOW
  rules:
  - to:
    - operation:
        methods: ["GET"]
        paths: ["/get"]
EOF
```

### 6. Test access control

```bash
# Allowed: GET /get
oc --context east exec -n bookinfo "$PRODUCTPAGE_POD" -- python3 -c "
import urllib.request
try:
    req = urllib.request.Request('http://httpbin.org/get')
    with urllib.request.urlopen(req, timeout=10) as resp:
        print(f'GET /get: {resp.status}')
except urllib.error.HTTPError as e:
    print(f'GET /get: {e.code} — {e.read().decode()[:50]}')
"

# Denied: GET /headers
oc --context east exec -n bookinfo "$PRODUCTPAGE_POD" -- python3 -c "
import urllib.request
try:
    req = urllib.request.Request('http://httpbin.org/headers')
    with urllib.request.urlopen(req, timeout=10) as resp:
        print(f'GET /headers: {resp.status}')
except urllib.error.HTTPError as e:
    print(f'GET /headers: {e.code} — {e.read().decode()[:50]}')
"

# Denied: POST /post
oc --context east exec -n bookinfo "$PRODUCTPAGE_POD" -- python3 -c "
import urllib.request
try:
    req = urllib.request.Request('http://httpbin.org/post', data=b'test', method='POST')
    with urllib.request.urlopen(req, timeout=10) as resp:
        print(f'POST /post: {resp.status}')
except urllib.error.HTTPError as e:
    print(f'POST /post: {e.code} — {e.read().decode()[:50]}')
"
```

Expected:
- `GET /get`: **200** (allowed)
- `GET /headers`: **403** (`RBAC: access denied`)
- `POST /post`: **403** (`RBAC: access denied`)

### 7. Cleanup

```bash
oc --context east delete authorizationpolicy httpbin-egress-policy -n egress-control
oc --context east delete serviceentry httpbin-org -n egress-control
istioctl --context east waypoint delete --namespace egress-control egress-gateway
oc --context east delete namespace egress-control
```

### 8. Verify recovery

```bash
curl -s -o /dev/null -w "EAST: HTTP %{http_code}\n" http://bookinfo.apps.cluster-64k4b.64k4b.sandbox5146.opentlc.com/productpage
```

Expected: HTTP 200, normal latency.

## What is Service Mesh here

| Component | Role | Mesh feature? |
|-----------|------|:------------:|
| IstioCNI (dnsCapture) | Redirects pod DNS queries to ztunnel proxy | Yes — DNS interception |
| ztunnel (DNS proxy) | Resolves ServiceEntry hosts to auto-allocated VIPs | Yes — L4 data plane |
| ServiceEntry | Registers external host (httpbin.org) in mesh registry | Yes — traffic management (GA) |
| egress-gateway (waypoint) | Processes outbound traffic, enforces L7 policies | Yes — L7 data plane |
| AuthorizationPolicy | Controls which methods/paths are allowed to external service | Yes — security (GA) |
| istiod | Pushes ServiceEntry + AuthorizationPolicy config to proxies | Yes — control plane |

## Key Takeaway

Egress control in ambient mode uses a **dedicated waypoint as an egress gateway**, combined with `ServiceEntry` and DNS capture. This is different from sidecar mode where each sidecar intercepts outbound traffic directly. The key enabler is **DNS capture** (`cni.ambient.dnsCapture: true`): without it, ztunnel cannot intercept traffic to external services because pods resolve DNS to real IPs that bypass the mesh. With DNS capture enabled, ztunnel resolves external hostnames to auto-allocated VIPs, routes traffic through the egress waypoint (via HBONE), and the waypoint enforces `AuthorizationPolicy` — providing fine-grained L7 control over which pods can access which external endpoints, methods, and paths.
